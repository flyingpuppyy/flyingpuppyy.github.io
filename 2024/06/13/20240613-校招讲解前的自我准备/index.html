cmd

命令提示符是在操作系统中，提示进行命令输入的一种工作提示符。在不同的操作系统环境下，命令提示符各不相同。 在windows环境下，命令行程序为cmd.exe，是一个32位的命令行程序，微软Windows系统基于Windows上的命令解释程序，类似于微软的DOS操作系统。输入一些命令，cmd.exe可以执行，比如输入shutdown -s就会在30秒后关机。总之，它非常有用。打开方法：开始-所有程序-附件 或 开始-寻找-输入：cmd/cmd.exe 回车。它也可以执行BAT文件。

powershell

PowerShell 是一种跨平台的任务自动化和配置管理框架，由命令行管理程序和脚本语言组成。 与大多数接受并返回文本的 shell 不同，PowerShell 构建在 .NET 公共语言运行时 (CLR) 的基础之上，接受并返回 .NET 对象。 这一根本上的改变引入了全新的自动化工具和方法。

PowerShell是跨平台的，cmd是Windows专用的

cmd是进入dos系统的命令，输入cmd进入的是dos系统，可以进行命令提示符操作

命令提示符窗口（cmd.exe）是Windows的“标配”组件，它可以实现用户与操作系统的直接交流，并负责用户输入的所有命令的解释和支持。

powershell是系统工具，是对cmd（命令提示符）的扩展，其中多了很多操作，比如管道操作。

PowerShell可以看作是cmd的升级版、补充版，但其背后的实现方式却有很大的不同，PowerShell基于面向对象的，更为易用。

PowerShell是运行在Windows操作系统上实现对系统以及应用程序进行管理自动化的命令行脚本环境，

PowerShell需要.NET环境的支持。

PowerShell从Windows7开始已经内置于Windows操作系统中

PowerShell有面向对象的管道。

PowerShell能够调用.NET的很多功能。

时间复杂度分析

最坏情况

最坏情况下，每次选择的基准值都是数组的最大或最小值，这样每次分区后，只有一个元素在基准值的一侧，而所有其他元素都在另一侧。这种情况的递归深度为 O(n)O(n)O(n)，每次分区操作的时间复杂度为 O(n)O(n)O(n)。因此，最坏情况的时间复杂度是： O(n2)O(n^2)O(n2)

具体步骤：

  1. 分区过程：每次分区操作需要 O(n)O(n)O(n) 次比较和交换。
  2. 递归深度：最坏情况下的递归深度是 nnn，因为每次只分出一个元素。

最好情况

在最好的情况下，每次选择的基准值都将数组均匀地分成两部分（即每次都能正好平分数组）。这种情况下，递归的深度为 O(log⁡n)O(logn)，每次分区的时间复杂度为 O(n)O(n)。因此，最好情况的时间复杂度是：
O(nlog⁡n)O(nlogn)

具体步骤：

分区过程：每次分区操作需要 O(n)O(n) 次比较和交换。
递归深度：最好情况下的递归深度是 O(log⁡n)O(logn)，因为每次都能平分数组。

双向链表的好处

双向遍历:

  • 正向遍历: 可以从头节点开始，按照顺序遍历到尾节点。
  • 反向遍历: 可以从尾节点开始，按照顺序遍历到头节点。这在需要从末尾到开头访问元素的应用中非常有用，如在浏览器的前进和后退操作中。

节点插入和删除:

  • 高效删除: 在给定节点本身的情况下，删除操作不需要遍历链表。只需调整被删除节点前后节点的指针即可完成。
  • 高效插入: 插入一个节点只需要调整相邻节点的指针，无需遍历链表，尤其是在中间位置进行插入时。这比单向链表更高效，因为在单向链表中插入节点通常需要找到前一个节点。

更灵活的链表操作:

  • 操作复杂性降低: 双向链表能够使某些操作更直观和简单。例如，反转链表的操作在双向链表中可以更加直接地实现。
  • 头尾操作的均衡性: 在双向链表中，操作头节点和尾节点同样高效。在某些需要频繁对头尾节点进行操作的场景中，双向链表是更好的选择。
