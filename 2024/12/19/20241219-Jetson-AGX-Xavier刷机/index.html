    当虚拟机网络设置为桥接模式时，虚拟机与主机以及局域网其他设备的 IP 关系如下：

    桥接模式网络的特点

    1. 虚拟机直接连接到物理网络
      • 虚拟机的网络适配器通过主机的物理网卡直接连接到局域网。
      • 虚拟机会像局域网中的一台独立设备，拥有自己的 IP 地址。
    2. IP 分配
      • 虚拟机通过 DHCP 或静态配置方式，获取与主机相同网段的 IP 地址。
      • 例如，主机 IP 是 192.168.1.100，虚拟机也可能获得类似 192.168.1.101 的 IP 地址。
    3. 虚拟机和主机的网络关系
      • 主机和虚拟机之间是同一网段，可以直接通信。
      • 虚拟机可以与局域网中的其他设备互通，就像局域网内的一台普通计算机。
    4. MAC 地址
      • 虚拟机的网络适配器会有自己的虚拟 MAC 地址，和主机的物理网卡区分开。

    主机与虚拟机之间的 IP 关系

    • IP 地址的分配：
      • 主机和虚拟机通常由局域网的 DHCP 服务器分配不同的 IP 地址。
      • 例如：
        • 主机 IP：192.168.1.100
        • 虚拟机 IP：192.168.1.101
    • 主机与虚拟机之间通信：
      • 主机可以直接通过虚拟机的 IP 地址访问虚拟机。
      • 比如，在主机上 ping 192.168.1.101 可以与虚拟机通信。
    • 虚拟机与其他设备通信：
      • 虚拟机可以访问主机、其他虚拟机以及局域网中的其他设备（如打印机、服务器等）。
